My Github Flow Command Line Cheatsheet

Github Flow is a development workflow that works well for data analysis projects as well as for small teams and individuals. On the command line, Github Flow uses git and their Go-based program hub which provides wrappers around common tasks.

The steps of Github Flow are:

  1. Create a branch
  2. Make commits
  3. Open a pull-request
  4. Merge (when ready)

hub installation instructions tell the user to set git as an alias for hub. hub will be used explicitly to show what is common git and what is specific to Github.

Getting Starting

Initial download of repository.

git clone https://github.com/USERNAME/REPONAME.git

With hub

hub clone USERNAME/REPONAME

Create a branch

Create a branch and switch to it.

git checkout -b BRANCHNAME

Push the new branch to Github so changes are tracked and synced.

git push origin BRANCHNAME:BRANCHNAME
git push --set-upstream origin BRANCHNAME

Switch back to the master

git checkout master

Work on a branch again

git checkout BRANCHNAME

Make commits

When ready to commit a step:

git add FILENAME
git commit -m 'description of change'
git push

Get list of commits on current branch but not the master. Prints the commit id and commit messages. Source

git cherry -v master

If you make major changes in one commit and want to go back to a previous commit, the easiest approach is creating a new branch from that old commit. Source

git checkout -b COMMITID

Create a pull request

Create a pull request for final review of changes.

hub pull-request

The pull-request url will be outputed to the console.

At this point, review and discussion can take place on Github.

Merge

After review, merge the branch onto the master.

hub am -3 https://github.com/USERNAME/REPONAME/pull/PULLNUM

The -3 flag specifies three-way merge.

Delete a branch

Source

After a merge, first tag it as archived, then delete the branch.

git tag archive/BRANCHNAME BRANCHNAME
git branch -d BRANCHNAME
git checkout master

To recover later.

git checkout archive/BRANCHNAME
git checkout -b NEWBRANCHNAME
